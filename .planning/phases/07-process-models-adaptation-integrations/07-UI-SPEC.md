## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, WIP badge inner padding |
| sm | 8px | Compact element spacing, toggle row gaps, switch label gaps |
| md | 16px | Default card padding, form field spacing, settings section padding |
| lg | 24px | Section padding, between settings panels |
| xl | 32px | Layout gaps between page sections |
| 2xl | 48px | Major section breaks (e.g. between /admin sub-sections) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets on toggle switches and admin action buttons: minimum 44px height (accessibility)
- Methodology card click target in project creation: minimum 80px height to accommodate icon + label + description
- WIP limit badge on Kanban column header: 24px height (compact indicator, not a primary action)

## Color

All color values reference the CSS custom properties defined in `Frontend/app/globals.css`. CSS variables are the source of truth; hex approximations are for executor reference only.

| Role | CSS Variable | Approx Value | Usage |
|------|-------------|--------------|-------|
| Dominant (60%) | `--background` | oklch(0.97 0.005 264) ≈ #f4f4f8 light | Page background, AppShell background, admin page background |
| Secondary (30%) | `--card` / `--sidebar` | oklch(1 0 0) = #ffffff light | Settings tab card surfaces, template list cards, admin panels, form surfaces |
| Accent (10%) | `--primary` | oklch(0.55 0.22 264) ≈ indigo-600 | See reserved list below |
| Destructive | `--destructive` | oklch(0.55 0.22 25) ≈ #dc2626 red | Delete custom template button, methodology change confirmation destructive variant |

**Accent (`--primary`) reserved for:**
- "Şablon Oluştur" primary button on `/admin/process-templates`
- "Değişiklikleri Kaydet" primary button in `/admin/settings`
- "Kaydet" primary button in Project Settings "Süreç Modeli" section
- Active methodology card selection ring/border (`ring-2 ring-primary`)
- "Bağlantıyı Test Et" button in project integrations section
- Admin nav sidebar active item indicator (`bg-sidebar-accent`)
- Module toggle active state (Switch `checked` thumb color via shadcn default — inherits `--primary`)

**WIP limit visual states (Kanban column header badge):**
- At or under limit: `bg-muted text-muted-foreground` (neutral, no alarm)
- At limit (= wip_limit): `border-amber-500 text-amber-700 bg-amber-50` + amber badge + `AlertTriangle` icon (h-3 w-3)
- Over limit (≥ wip_limit + 2): `border-red-500 text-red-700 bg-red-50` + red badge + `AlertTriangle` icon

**Methodology card color coding (project creation + template list):**
- No unique color per methodology — all use the same card surface with accent ring on selection
- Active selected card: `ring-2 ring-primary border-primary`
- Inactive card: `border-border hover:border-primary/50`

## Layout Contract

### `/admin` Route Space

Admin-only. Protected by role check (`role.name.toLowerCase() === 'admin'`). Redirect non-admins to `/` with a "Yetkisiz erişim" toast.

Admin pages wrap in `<AppShell>`. Navigation between sub-pages via `Tabs` at the top of the admin layout (not a separate sidebar):

```
[AppShell]
  [Page header row]                  ← flex justify-between items-center mb-6
    h1: "Yönetim Paneli"             ← text-2xl font-bold (Display size)
    [Admin badge]                    ← Badge variant="secondary": "Admin"

  [Admin Tabs nav]                   ← Tabs defaultValue="process-templates"
    TabsList
      TabsTrigger: "Süreç Şablonları"  ← value="process-templates"
      TabsTrigger: "Sistem Ayarları"   ← value="settings"

    TabsContent value="process-templates"
      → /admin/process-templates content (see below)

    TabsContent value="settings"
      → /admin/settings content (see below)
```

```
[Section header row]                 ← flex justify-between items-center mb-4
  h2: "Süreç Şablonları"             ← text-xl font-semibold

  Columns: Şablon Adı | Tür | Kolon Sayısı | Eylemler
  Built-in rows: read-only; "Yerleşik" Badge (variant="secondary"), Eye icon, no edit/delete
  Custom rows: "Özel" Badge (variant="outline"), Edit button (ghost), Delete button (ghost, destructive color)
```

## Interaction Contracts

### Methodology Selection (Project Creation)

| Interaction | Behavior |
|-------------|----------|
| Click methodology card | Card gets `ring-2 ring-primary border-primary`. Board columns preview updates below to show template columns. |
| ITERATIVE card selected | Shows columns: Planlama, Analiz, Geliştirme, Değerlendirme |
| Submit form without methodology selected | "Bir süreç modeli seçin." validation error below the card grid. Button stays disabled. |

### Methodology Change (Project Settings)

| Interaction | Behavior |
|-------------|----------|
| Change Select value to a different methodology | `ConfirmDialog` opens immediately before saving |
| Confirmation dialog | Title: "Süreç Modelini Değiştir". Body: "Mevcut görevler ve board kolonları korunacak. SCRUM'dan geçiş yapıyorsanız aktif sprint'ler kapatılacak. Devam etmek istiyor musunuz?" [İptal] [Değiştir (primary)] |
| Confirm | PATCH /projects/{id} fires. Spinner on "Kaydet" button. Sprints archived server-side if switching away from SCRUM. process_config updated. |
| Success | sonner toast: "Süreç modeli güncellendi." Page data refetches. |
| Error | sonner toast: "Süreç modeli güncellenemedi. Lütfen tekrar deneyin." Select reverts to previous value. |

### Behavioral Flag Toggles

| Interaction | Behavior |
|-------------|----------|
| Toggle a flag Switch | Optimistic local update. PATCH /projects/{id} with updated process_config. |
| PATCH success | No toast (silent save). Tooltip confirms current state. |
| PATCH error | Switch reverts. sonner toast: "Ayar kaydedilemedi." |

### WIP Limit Toast (Kanban Board)

| Interaction | Behavior |
|-------------|----------|
| Drop task into column that exceeds wip_limit | sonner toast (warning, non-blocking): "Uyarı: Bu kolon için belirlenen WIP (Aynı Anda Yapılan İş) limiti aşıldı!" Duration: 5000ms. |
| Column header badge update | Immediate (optimistic) — updates based on current task count vs wip_limit after drop. |
| Drop still succeeds | Task moves to column regardless (warning-only, not a hard block per CONTEXT.md D-06). |

### Custom Template CRUD (Admin)

| Interaction | Behavior |
|-------------|----------|
| Click "Şablon Oluştur" | Dialog opens with blank form. |
| Submit create form | POST /process-templates. Button shows Loader2 spinner. Dialog closes on success. Table refetches. sonner toast: "Şablon oluşturuldu." |
| Click Edit on custom template | Same Dialog opens pre-populated. |
| Submit edit form | PATCH /process-templates/{id}. Same optimistic spinner pattern. sonner toast: "Şablon güncellendi." |
| Click Delete on custom template | AlertDialog: "Bu şablonu silmek istediğinize emin misiniz? Bu işlem geri alınamaz." [İptal] [Sil (destructive)] |
| Confirm delete | DELETE /process-templates/{id}. AlertDialog closes. Table refetches. sonner toast: "Şablon silindi." |
| Click Eye on built-in template | No action (read-only). Cursor: `cursor-default`. Tooltip: "Yerleşik şablonlar düzenlenemez." |

### Admin System Config Save

| Interaction | Behavior |
|-------------|----------|
| Edit any system parameter field | Form becomes "dirty" (button enables). |
| Click "Değişiklikleri Kaydet" | PUT /admin/settings. Button shows Loader2 spinner + "Kaydediliyor..." text. |
| Success | sonner toast: "Ayarlar kaydedildi. Değişiklikler hemen uygulandı." Button returns to default. |
| Error | sonner toast: "Ayarlar kaydedilemedi. Lütfen tekrar deneyin." Fields retain edited values. |

### Module Toggle (Reporting Module)

| Interaction | Behavior |
|-------------|----------|
| Toggle Switch off | Immediate PATCH /admin/settings. "Raporlar" nav item disappears from sidebar (conditional render based on SYSTEM_CONFIG). |
| Toggle Switch on | "Raporlar" nav item re-appears. |
| State reads from | SYSTEM_CONFIG query on mount. No page reload required. |

### Webhook Connection Test

| Interaction | Behavior |
|-------------|----------|
| Click "Bağlantıyı Test Et" | Button shows Loader2 spinner. POST to backend test endpoint with current webhook URL. |
| URL field empty | Button is `disabled`. Tooltip: "Önce webhook URL girin." |
| Success response | Inline: `CheckCircle2 className="text-green-600"` + "Bağlantı başarılı!" replaces spinner inline. Duration: stays visible until next user action. |
| Failure response | Inline: `XCircle className="text-destructive"` + "Bağlantı başarısız. URL'yi kontrol edin." |

### Admin Route Guard

| Interaction | Behavior |
|-------------|----------|
| Non-admin user visits /admin/* | Redirect to `/`. sonner toast: "Bu sayfaya erişim yetkiniz yok." |
| Admin master integration switch turned off | All "Bağlantıyı Test Et" buttons across all projects are disabled (read from SYSTEM_CONFIG context). Backend rejects webhook sends silently. |
