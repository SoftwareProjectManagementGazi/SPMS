### Surface F — `<DataState/>` 3-state primitive

**File:** `Frontend2/components/primitives/data-state.tsx` (NEW). **Adopted by:** every chart card, activity timeline, profile tab content, Faz Raporları section. **Optional retroactive adoption** in dashboard ActivityFeed empty state (Plan 13-03).

#### F.1 Hierarchy & Layout

```tsx
interface DataStateProps {
  loading?: boolean
  loadingFallback?: React.ReactNode
  error?: Error | null | unknown
  errorFallback?: React.ReactNode | ((err: unknown, retry?: () => void) => React.ReactNode)
  empty?: boolean
  emptyFallback?: React.ReactNode
  onRetry?: () => void
  children: React.ReactNode
}

// Render priority: error > loading > empty > children
```

DataState is layout-neutral — it renders ONLY one of {loadingFallback, errorFallback, emptyFallback, children}. Caller is responsible for the surrounding card / heading. No padding / margin / box added by the primitive itself.

| Property | Value |
|----------|-------|
| Wrapper element | `<>` (Fragment — zero DOM) |
| Default loadingFallback | `<DataStateDefaultLoading/>` — centered "Yükleniyor…" 40px vertical pad, --fg-muted, fontSize 13 |
| Default errorFallback | `<AlertBanner tone="danger" action={onRetry && <Button size="sm" onClick={onRetry}>Tekrar dene</Button>}>{err?.message ?? "Veri alınamadı."}</AlertBanner>` |
| Default emptyFallback | `<DataStateDefaultEmpty/>` — centered "Veri yok." 40px vertical pad, --fg-muted, fontSize 13 |

#### F.2 Typography

Default fallbacks use:
- Loading: 13 / 400 / sans / --fg-muted
- Error AlertBanner: 12.5 / 400 (per AlertBanner primitive)
- Empty: 13 / 400 / sans / --fg-muted

#### F.3 Color

Defaults:
- Loading: --fg-muted
- Error: --priority-critical (via AlertBanner tone="danger")
- Empty: --fg-muted

#### F.4 Interaction

| Trigger | Action |
|---------|--------|
| `error` prop becomes truthy | Render errorFallback. If function form, callback receives `(error, onRetry)` |
| `loading` true & no error | Render loadingFallback |
| `empty` true & not loading & no error | Render emptyFallback |
| All states false | Render children |
| Click default error retry Button | Invoke `onRetry()` if passed |

The primitive does NOT manage its own state; it's a pure render switch. Prop priority order is documented inline as a comment.
