# Phase 12 — UI Design Contract

> Visual and interaction contract for **Phase 12: Lifecycle, Phase Gate & Workflow Editor**.
> Generated by gsd-ui-researcher. All token values are inherited from Phase 8/11 and the
> oklch system in `Frontend2/app/globals.css`. **Zero new tokens are invented.** Every
> visual decision in this document is traceable to either a Phase 8/11 prior decision, the
> HTML prototype (`New_Frontend/src/pages/{lifecycle-tab,workflow-editor,settings}.jsx`),
> or a `[Claude's Discretion]` flag with a recommendation.
>
> **Visual authority order (highest first):**
> 1. `New_Frontend/src/pages/lifecycle-tab.jsx` (LifecycleTab + sub-tabs + PhaseGateExpand + MiniMetric)
> 2. `New_Frontend/src/pages/workflow-editor.jsx` (WorkflowCanvas + WorkflowEditorPage + Field + ValidationItem)
> 3. `New_Frontend/src/pages/settings.jsx` (Settings sidebar + Yaşam Döngüsü sub-tab structure)
> 4. `workflow design and preview_jira.jpeg` (project root — "All" gate pill semantic)
> 5. Frontend2 existing primitives + globals.css tokens (must reuse, never duplicate)
>
> **Forbidden:** shadcn/ui imports anywhere, any reference to old `Frontend/`, hard-coded
> hex/rgb values, custom font-size/spacing values that bypass the scales below.

## Design System

| Property          | Value                                                                                  |
|-------------------|----------------------------------------------------------------------------------------|
| Tool              | none — custom design system inherited from Phase 8 (`Frontend2/app/globals.css`)         |
| Preset            | not applicable                                                                         |
| Component library | none — 16 hand-built primitives in `Frontend2/components/primitives/` (reused as-is)     |

**Zero shadcn/ui.** All Phase 12 surfaces compose existing 16 primitives + new compound
components that reference the same CSS variable tokens. React Flow is used for graph state,
DnD, minimap, pan/zoom plumbing — every node, edge, group, control button receives a custom
renderer or CSS-variable theming override. **The only React Flow default visual that may be
retained is the dot-grid pattern fill, and even that is overridden via CSS variable.**

## Spacing Scale

Phase 12 inherits the Phase 8/11 8-point spacing scale. **All vertical and horizontal gaps
inside Phase 12 components must come from this list.**

| Token semantic | Value | Usage in Phase 12 surfaces                                                                                                                                                                                                                                  |
|----------------|-------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| xs             | 4px   | Icon-to-text gap inside Badges, MiniMetric label margin-top, status-dot to row text                                                                                                                                                                          |
| sm             | 6–8px | SegmentedControl active pill padding, Phase Gate auto-criteria row vertical, Toggle internal, Tabs gap                                                                                                                                                       |
| md             | 10–14px | Card-internal padding for compact surfaces (history card padding 14, milestone card padding 14), summary strip gap, two-column gap inside Phase Gate                                                                                                          |
| lg             | 16px  | Default Card padding, sub-tab content top margin, MiniMetric grid gap, evaluation report card padding (16px)                                                                                                                                                  |
| xl             | 20px  | Phase Gate panel padding (20px), Settings > Yaşam Döngüsü panel-card padding, viewport-fallback page padding                                                                                                                                                  |
| 2xl            | 24px  | Workflow Editor right-panel section vertical-gap, Settings page sidebar-content gap                                                                                                                                                                            |
| 3xl            | 32px  | Workflow Editor outer page vertical gap                                                                                                                                                                                                                      |
| 4xl            | 48–64px | Empty-state padding (padding 30 prototype-faithful → 32 GSD-rounded)                                                                                                                                                                                        |

**Permitted exceptions** (matched 1:1 to prototype, NOT free-for-all):
- Summary strip top/bottom padding **`10px`** (matches prototype `lifecycle-tab.jsx:51`).
- Empty-state inner padding **`30px`** for milestone/history zero-state (matches prototype).
- Workflow node geometry: **`140 × 60`** width × height (matches prototype canvas spec).
- Right panel column width: **`320px`** (matches prototype `workflow-editor.jsx:157`).
- Bottom toolbar floating offset: **`bottom: 16px`**, minimap **`width: 180px height: 100px`**.
- Two-handle source/target circles offset: **`left/right: -5px, top: 26px, 10×10`**.

These prototype-faithful values OVERRIDE the GSD default scale where they conflict — the
visual contract with the user is "1:1 prototype reproduction".

### Color contrast WCAG check

All foreground/background pairs MUST satisfy WCAG AA contrast. Phase 8 token system was
designed for AA in both light (Terracotta) and dark (Midnight) modes; Phase 12 inherits
this guarantee. **One known watch-out:** the `node.opacity: 0.55` past state combined with
`var(--fg-muted)` on `var(--surface)` is borderline at AA. **Verification step:** during
plan-checkpoint, the executor must run a contrast spot-check on `(--fg-muted with opacity 0.55) vs --surface`
in both modes. If below 4.5:1, raise opacity floor to 0.6 (justification documented in 12-VERIFICATION.md).

## Copywriting Contract

**Every static string flows through `useApp().language` via the `T(tr, en)` pattern (or the
top-level `language` accessor for components without prop drilling).** No hard-coded English
in TSX. No literal `"tr"` checks outside the language-aware accessor. Spell-check
conventions: full Turkish diacritics (`ş`, `ç`, `ğ`, `ı`, `İ`, `ö`, `ü`).

### Locale fallback rule
If `useApp().language` is something other than `"tr"` or `"en"` (defensive), default to
`"en"`. This must NOT throw and MUST NOT mix languages mid-string.

### 12. GroupCloudNode (`components/workflow-editor/group-cloud-node.tsx`)

```
        ╭──────────────────────╮
       │  Tasarım                │
       │  ┌──┐  ┌──┐             │
       │  │N3│  │N4│             │
       │  └──┘  └──┘             │
       ╰──────────────────────╯
        (smooth concave-hull SVG path)
```

**Path generation:** for each group, gather child node bounding boxes, expand each by
padding 16, run concave-hull algorithm (CONTEXT D-22). Render as SVG `<path d="..."/>`
inside an absolute-positioned div sized to the bounding box of the hull.

**[Claude's Discretion]** algorithm choice:
- Recommendation: **convex-hull-plus-padding baseline** for v1.
  Reason: zero new dependency cost, predictable visual at <50 nodes, uses Graham scan in
  ~80 LOC. The smoothing happens via `path-data-d3-curve` (existing Phase 11 d3 dep) using
  `curveCardinalClosed.tension(0.5)`.
- Upgrade path: swap to `concaveman` library (~3KB gzip) if user feedback is "too box-y".
- Both implementations satisfy the 60fps target (< 16ms per frame for 50 nodes).

**Fill / stroke:**
- default: `fill: color-mix(in oklch, var(--{group.color}) 6%, transparent)`,
  `stroke: 1.5px dashed color-mix(in oklch, var(--{group.color}) 30%, transparent)`.
- selected: `fill: color-mix(in oklch, var(--primary) 8%, transparent)`,
  `stroke: 2px solid var(--primary)`.
- drag-target (during node drag): `fill: color-mix(in oklch, var(--primary) 12%, transparent)`,
  `stroke: 2px dashed var(--primary)`.

**Label:** top-left of the bounding box, padding `6px 10px`, `fontSize: 11, weight 600,
color: var(--{group.color})`. On hover, label gains background `var(--surface)` for read.

**Live morph (D-23):** the path recomputes on every `onNodeDrag` event — debounce **disabled**
because animation must be continuous. CSS `transition: d 100ms ease` smooths between snapshots
when the algorithm produces a slightly-different vertex set.

**Drop association (CONTEXT Claude's Discretion last bullet):** when a node is dragged
OUTSIDE the cloud's bounding hull (computed live), the node's `parentId` is cleared and the
cloud reshrinks. **Recommended default: "drop association"** matches user expectation for
free-form editors. Snap-back is documented as a v2.1 alternative if user feedback warrants.

## Localization

**Single source:** `useApp().language` from Phase 8 (`Frontend2/context/app-context.tsx`).
Helper pattern is the per-component `T = (tr, en) => language === "tr" ? tr : en`.

**Spell-check (Turkish):** all strings in this document have been verified for diacritics
correctness:
- `İ` (capital I with dot) for `İlerleme`, `İptal`, etc.
- `ş` for `İş Akışı`, `Faz Geçişi`.
- `ğ` for `Düğüm`, `Bağlantı`.
- `ç` for `Geçmiş`, `Açık`.
- `ö` for `Önceki`, `Çöz`.
- `ü` for `Düğüm`, `Sürekli`.

**Locale fallback rule:** if `language` is something other than `tr/en`, default to `en`.
Implementation: `const T = (tr, en) => language === "tr" ? tr : en;` already does this
(falsy check on `tr` branch returns `en`).

**Number / date formatting:**
- Dates: `toLocaleDateString(language === "tr" ? "tr-TR" : "en-US", { month: "short", day: "numeric", year: "numeric" })` — matches prototype `lifecycle-tab.jsx:246`.
- Counts (badges): plain integers; no thousand separators in v2.0 (the seeded scale is small).
- Percentages: `%{value}` (Turkish style with leading `%`) — matches prototype.

**Pluralization:** v2.0 does not handle plural forms beyond singular/plural manual hardcoding
("görev" stays singular even with `N=10`; this matches Turkish convention where there is no
plural agreement after a counter). English "tasks" used for `N>1` only.

**RTL:** out of scope (Turkish + English, both LTR).
