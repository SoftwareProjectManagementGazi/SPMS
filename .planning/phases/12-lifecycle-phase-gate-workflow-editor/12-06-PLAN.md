## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to PATCH /artifacts/{id} | Manager-only path (Phase 9 D-36) |
| Browser to PATCH /artifacts/{id}/mine | Assignee-only path (Phase 9 D-36) |
| Browser to GET /phase-reports/{id}/pdf | 30s rate-limited per-user (Phase 9 D-51) |
| Browser to file upload | Uses Phase 3 upload machinery; Phase 9 D-41 single-file constraint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06-01 | E (Elevation of Privilege) | Artifact PATCH route selection | mitigate | Frontend chooses `/mine` vs base path based on `user.id === artifact.assignee_id` (Phase 9 D-36 + STATE.md line 162). Backend re-validates: `/mine` rejects non-assignee, base path rejects non-PM. Defense-in-depth via `useTransitionAuthority` for delete. |
| T-12-06-02 | D (Denial of Service) | PDF rate limit | mitigate | Frontend honors 429 with countdown toast "30 saniye bekleyin" (CONTEXT D-58). UI prevents repeated clicks during loading via disabled button + spinner. |
| T-12-06-03 | T (Tampering) | revision field | mitigate | Frontend NEVER sends `revision` in PATCH body; backend auto-increments via Phase 9 D-25 + STATE.md line 150 (revision auto-increment). UpdateDTO at the service layer omits the field entirely. |
</threat_model>
