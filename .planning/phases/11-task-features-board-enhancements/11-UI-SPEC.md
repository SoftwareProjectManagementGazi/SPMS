# Phase 11 — UI Design Contract

> Visual and interaction contract for Phase 11: task-features-board-enhancements.
> Generated by gsd-ui-researcher. All token values extracted from New_Frontend/src/styles.css,
> New_Frontend/src/primitives.jsx, and the prototype page files. No values invented.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom design system from New_Frontend prototype |
| Preset | not applicable |
| Component library | none (16 hand-built primitives in Frontend2/components/primitives/) |
| Icon library | Lucide React (used as Icons.* in prototype) |
| Font | Geist (sans) + Geist Mono (mono) |

**Zero shadcn/ui.** All overlays, modals, popovers, dropdowns built from the 16 existing
primitives (Avatar, AvatarStack, Badge, Button, Card, Kbd, Tabs, Section, PriorityChip,
StatusDot, Input, ProgressBar, SegmentedControl, Collapsible, AlertBanner, Toggle) plus new
compound components that use the same CSS variable tokens.

### Avatar Color Tokens

`--av-1` through `--av-8` — 8 distinct oklch hues. Assigned via `user.avColor` index.

### WIP Violation Color Formula

Column bg when over WIP limit: `color-mix(in oklch, var(--priority-critical) 6%, var(--bg-2))`
Column bg when at WIP limit (warning): `color-mix(in oklch, var(--status-review) 4%, var(--bg-2))`

### Label Color Formula

`hue = hash(labelName) % 360`
Badge bg: `oklch(0.70 0.13 ${hue})`
Badge text (auto-contrast): if hue lightness > 0.65 use `oklch(0.20 0.025 50)` else `oklch(0.985 0.005 75)`
Implementation: `lib/label-color.ts` — pure function, no CSS at runtime.

## Color Contract

| Role | Value | 60/30/10 Allocation |
|------|-------|---------------------|
| Dominant (60%) | `--bg` + `--surface` | Page bg, card fill, modal fill, calendar cells, board card bg |
| Secondary (30%) | `--bg-2` + `--surface-2` | Board column bg, table row hover, input bg, segment inactive, sidebar, group headers |
| Accent (10%) | `--primary` | Active tab underline, active SegmentedControl pill bg, primary button, today circle, watcher toggle-on |

**Accent reserved for:**
- Active tab underline (2px solid `--primary`)
- Primary CTA button fill
- Today's date circle in Calendar cells
- Current sprint badge tint (`tone="info"` maps to `--status-progress`)
- Focus ring (`--ring`)
- Active saved-view tab bg uses `--accent` (not `--primary` — softer)
- Watcher button "Takipte" fill via `tone="primary"` Badge

**Destructive color (`--priority-critical`):**
- Danger Button variant only
- WIP violation column tint (`6%` mix)
- Overdue due-date chip color
- Critical priority chip
- Comment delete confirmation button
- Dependency remove button (hover state only)

**Methodology badge tones** (per prototype):
- Scrum: `tone="info"` → `--status-progress` tint
- Kanban: `tone="primary"` → `--primary` tint
- Waterfall + others: `tone="warning"` → `--status-review` tint

## Accessibility Contract

### Focus Management

- **Task Create Modal:** On open, `autoFocus` on Title input. Focus trapped inside modal (Tab cycles within). On close (Esc or Vazgeç or successful submit), focus returns to the trigger element (`header Create button` or `project "Görev" button`).
- **InlineEdit:** On click-to-edit, focus moves to the edit control. On commit or cancel, focus returns to the row element.
- **Backlog panel toggle:** Toggle button maintains focus after click. Panel open/close does not steal focus from currently active element unless user explicitly clicked the toggle.
- **Dropdown/pickers (MTPicker, status picker, search dropdown):** Open on click/Enter, trap focus within dropdown while open, close on Escape (focus returns to trigger), close on click-outside.
- **Page navigation (board card click, task row click):** Standard `router.push` — browser manages focus on navigation.

### Keyboard Shortcuts Inventory

| Shortcut | Action | Scope |
|----------|--------|-------|
| Cmd/Ctrl + K | Focus Header search input | Global |
| Cmd/Ctrl + Enter | Submit Task Create Modal | Modal open |
| Escape | Close Task Create Modal / Close dropdowns / Close popover / Cancel InlineEdit | Context-sensitive |
| Arrow Down | Move to next result in Search dropdown | Search open |
| Arrow Up | Move to previous result in Search dropdown | Search open |
| Enter | Navigate to highlighted search result | Search open |
| Arrow keys (Left/Right/Up/Down) | Move card between columns | Board with `@dnd-kit KeyboardSensor` |
| Space | Pick up / drop card | Board drag with `@dnd-kit KeyboardSensor` |
| Tab | Cycle through InlineEdit fields in Properties sidebar | Properties sidebar |
| Enter | Commit InlineEdit | InlineEdit active |
| Escape | Cancel InlineEdit | InlineEdit active |

### ARIA Roles

- Tab bar (`Tabs` component): `role="tablist"`, each tab `role="tab"`, `aria-selected`, `aria-controls`.
- Modal overlay: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title id.
- Board columns: `role="list"` on column body, `role="listitem"` on each card.
- Backlog task list: `role="list"` / `role="listitem"`.
- Search dropdown: `role="listbox"`, each result `role="option"`, `aria-selected`.
- SegmentedControl: `role="radiogroup"`, each button `role="radio"`, `aria-checked`.
- AlertBanner: `role="alert"` when WIP violation appears dynamically.
- Properties sidebar MetaRow: `role="row"` not required (not a table); plain divs acceptable.
- Collapsible: trigger `aria-expanded`, content region `aria-hidden` when collapsed.
- Drag-and-drop: `@dnd-kit` provides built-in ARIA announcements via `announcements` prop. Configure: "Kart {task.key} alındı", "Kart {task.key} {columnName} kolonuna bırakıldı".

### Color Contrast

- All body text (`--fg` on `--surface`): meets WCAG AA (4.5:1+) in both light and dark modes.
- Muted text (`--fg-muted` on `--surface`): meets WCAG AA 4.5:1 in light mode. Verify in dark mode.
- Badge text: each tone's fg/bg pair was designed for contrast. Critical tone: verify `--priority-critical` bg with white fg.
- Priority chip labels `--fg-muted` on transparent bg: acceptable (decorative + has icon).
- Status dots: always paired with text label in list/sidebar contexts (not dot-only for status identification).
- Due date chips: `overdue` tone uses `--priority-critical` color — confirm 3:1 minimum on chip bg.
- Do NOT use color as the sole differentiator for status or priority — always pair with icon or label.

## Responsive Contracts

### Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| ≥ 1280px | Full layout: backlog panel + content, 2-col task detail, all toolbar items visible |
| < 1280px | Backlog panel auto-closes (ignores localStorage if `true`). Toggle button stays. Narrow-screen hint text shown in panel if user force-opens. |
| < 1024px | ProjectDetail 8-tab bar: `overflowX:auto` (scrollable, fade mask at edges). Task Detail: `gridTemplateColumns:1fr` — sidebar stacks below main as expandable sections. |
| < 1024px | Board toolbar: phase filter dropdown hidden (tooltip visible via icon). AvatarStack max:3 instead of 5. |

### Board at narrow widths

< 1280px: board grid `gridTemplateColumns: repeat({n}, minmax(240px, 1fr))` with `overflowX:auto` on wrapper. Cards readable but may require horizontal scroll.

### Task Detail sidebar < 1024px

Properties sidebar renders below main content. Full width. Each property row displays as `grid: 140px 1fr` (wider label column for readability). Sections (Dependencies, Attachments) are Collapsible components with defaultOpen:false.

### MyTasks

Full-width layout at all breakpoints (no right rail on < 1024px — `hideRightRail` prop used for compact). Toolbar wraps with `flexWrap:wrap`.

### Task Create Modal

Modal is `width:540px` with `maxHeight:85vh`. On < 600px viewport: `width:calc(100vw - 32px)`. Padding unchanged. 2-col grid inside becomes 1-col: `gridTemplateColumns: 1fr`.
