1. Define `ThemeTokens` interface with all token keys (bg, bg-2, surface, surface-2, fg, fg-muted, fg-subtle, border, border-strong, primary, primary-fg, primary-hover, accent, accent-fg, ring).

4. Export `TOKEN_MAP: Record<string, string>` mapping prototype token names to shadcn CSS variable names: `{ bg: "background", surface: "card", fg: "foreground", "fg-muted": "muted-foreground", "primary-fg": "primary-foreground", "accent-fg": "accent-foreground" }`. Tokens not in the map pass through with their original name.

5. Export `function applyTokens(tokens: ThemeTokens): void` that:
   - Iterates all token entries

   - Derives shadcn-specific tokens: `--popover` = tokens.surface, `--popover-foreground` = tokens.fg, `--card-foreground` = tokens.fg, `--input` = tokens.border, `--secondary` = tokens["surface-2"], `--secondary-foreground` = tokens.fg, `--muted` = tokens["surface-2"], `--sidebar` = tokens["bg-2"], `--sidebar-foreground` = tokens.fg, `--sidebar-primary` = tokens.primary, etc.

6. Export `function applyMode(mode: "light" | "dark"): void` that sets `document.documentElement.dataset.mode = mode`.

7. Export `function deriveFromBrand(params: { L: number; C: number; H: number; mode?: "light" | "dark" }): ThemeTokens` ported from `New_Frontend/src/theme.jsx` `deriveFromBrand()` function. 8. Export `function estimateContrast(l1: number, l2: number): number` ported from theme.jsx for contrast checking.

**context/app-context.tsx -- New file (per D-07, D-08):**

Create `Frontend/context/app-context.tsx` following the auth-context.tsx pattern (Pattern G from PATTERNS.md):

1. File header:
   ```typescript
   "use client"
   import * as React from "react"
   import { PRESETS, applyTokens, applyMode, deriveFromBrand } from "@/lib/theme"
   import type { LangCode } from "@/lib/i18n"
   ```

2. Define `AppContextType` interface:
   ```typescript
   type ThemeMode = "light" | "dark"
   type ThemePreset = "default" | "ocean" | "forest" | "monochrome" | "midnight" | "graphite"
   type Density = "compact" | "cozy" | "comfortable"
   
   interface AppContextType {
     language: LangCode
     setLanguage: (lang: LangCode) => void
     mode: ThemeMode
     setMode: (mode: ThemeMode) => void
     preset: string
     applyPreset: (id: string) => void
     sidebarCollapsed: boolean
     setSidebarCollapsed: (collapsed: boolean) => void
     density: Density
     setDensity: (density: Density) => void
     customColors: boolean
     brandHue: number
     brandChroma: number
     brandLight: number
     applyCustomBrand: (params: { L: number; C: number; H: number }) => void
   }
   ```

3. Create context: `const AppContext = React.createContext<AppContextType | undefined>(undefined)`

4. Export `AppProvider` component:
   - Initialize state from localStorage with defaults: language="tr" (D-06), mode="light", preset="default", sidebarCollapsed=false, density="cozy", customColors=false
   - On mount (`React.useEffect`), read persisted values from localStorage keys: `spms-language`, `spms-theme-mode`, `spms-theme-preset`, `spms-sidebar-collapsed`, `spms-density`, `spms-custom-brand`
   - `applyPreset(id)` function: looks up `PRESETS[id]`, calls `applyTokens(preset.tokens[mode])`, persists to localStorage
   - `setMode(mode)` function: calls `applyMode(mode)`, re-applies current preset tokens for the new mode, persists to localStorage
   - `setLanguage(lang)` function: updates state and persists to `spms-language`
   - `applyCustomBrand({ L, C, H })` function: calls `deriveFromBrand({ L, C, H, mode })`, then `applyTokens()` with result, sets customColors=true
   - On mount effect: apply initial preset tokens and mode so tokens are active from first render
   - Set `data-density` attribute on documentElement when density changes

5. Export `useApp()` hook with guard (same pattern as useAuth):
   ```typescript
   export function useApp() {
     const context = React.useContext(AppContext)
     if (!context) throw new Error("useApp must be used within AppProvider")
     return context
   }
   ```

| Boundary | Description |
|----------|-------------|
| localStorage -> AppContext | User preferences read from localStorage; no sensitive data |
| Backend config -> theme | primary_brand_color from API used in deriveFromBrand |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | localStorage theme prefs | accept | Theme preferences (language, mode, preset) are non-sensitive UI state. Tampering only affects the attacker's own view. No PII, no auth data. |
| T-08-02 | Information Disclosure | globals.css token values | accept | CSS custom properties are visible in browser DevTools by design. No sensitive data in token values. |
| T-08-03 | Tampering | data-mode attribute on html | accept | User can manually change dark mode attribute. This is cosmetic only -- no security impact. |
</threat_model>
