Component conversion rules (from RESEARCH.md Pattern 2):
1. Add "use client" directive as first line
2. Use `import * as React from "react"` (namespace access convention)
3. Create explicit Props interface with all props typed
4. Add className prop (missing from prototype) for Tailwind composition
5. Use `export function ComponentName` (named export convention, PascalCase)
6. Convert STATIC styles to Tailwind classes where clean mapping exists
7. Keep `style={}` for DYNAMIC values that depend on props (height, color, width%)
8. Keep `style={}` for complex token expressions (color-mix, compound boxShadow with var())
9. Use `cn()` from `@/lib/utils` for className merging
10. Keep `var()` CSS variable references for token values

Standard file template:
```typescript
"use client"

import * as React from "react"
import { cn } from "@/lib/utils"

interface ComponentNameProps {
  // typed props
  className?: string
}

export function ComponentName({ prop = default, className }: ComponentNameProps) {
  return (
    <div className={cn("tailwind-classes", className)} style={{ /* dynamic */ }}>
      {/* content */}
    </div>
  )
}
```
