<objective>
Build the backend foundation that all four view tabs depend on: a CRUD API for board columns (so the Kanban board can render dynamic columns), sprint close/delete with task reassignment (so the Phase management UI can manage phases), and a frontend `patchTask` method for lightweight partial updates on drag events.

Purpose: Without this plan, the Kanban board cannot fetch dynamic columns, the Phase management page cannot close/delete phases safely, and DnD drag events cannot update task column_id or due_date without sending the full task payload.

Output: 5 new backend files (column repository interface, SQLAlchemy repo, DTOs, use cases, router), updates to sprints use cases and router, update to task-service.ts.
</objective>

    **Sprint close/delete use cases** (`Backend/app/application/use_cases/manage_sprints.py`):
    Add `CloseSprintUseCase`:
    - `execute(sprint_id, move_tasks_to_sprint_id: int | None)` — sets `is_active=False` on the sprint, then updates all tasks where `sprint_id = sprint_id` AND task is not done: set `sprint_id = move_tasks_to_sprint_id` (null = backlog).
    - "Done" check: task's column is the last column (highest order_index) OR task.status == 'Done'. Use `SELECT tasks WHERE sprint_id = :sid AND column_id NOT IN (SELECT id FROM board_columns WHERE project_id = :pid ORDER BY order_index DESC LIMIT 1)` to find incomplete tasks.

    - Preferred approach: count tasks where their column's name != 'Done' (case-insensitive). Move only those.
