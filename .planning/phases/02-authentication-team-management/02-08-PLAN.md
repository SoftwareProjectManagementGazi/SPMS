**Frontend/services/auth-service.ts** — ADD two methods to the existing authService object:
```typescript
requestPasswordReset: async (email: string): Promise<void> => {
  await apiClient.post('/auth/password-reset/request', { email });
},

confirmPasswordReset: async (token: string, newPassword: string): Promise<void> => {
  await apiClient.post('/auth/password-reset/confirm', {

    new_password: newPassword,
  });
},
```

**Frontend/app/forgot-password/page.tsx** (new file):
- `"use client"`
- Single email input with submit button
- State: `email`, `isSubmitting`, `submitted: boolean`
- On submit: call `authService.requestPasswordReset(email)`, then set `submitted = true`
- After submit (success OR error from network): show generic message "If this email is registered, you'll receive a reset link shortly." — do NOT show different messages for registered vs unregistered email (no user enumeration — locked CONTEXT.md decision)

**Frontend/app/reset-password/page.tsx** (new file):
- `"use client"`
- Extract `token` from URL search params: `const searchParams = useSearchParams(); const token = searchParams.get('token')`
- If no token in URL: show "Invalid or missing reset link" with link to /forgot-password
- State: `newPassword`, `confirmPassword`, `isSubmitting`, `error: string | null`, `success: boolean`
- Validation before submit: `newPassword === confirmPassword`, minimum length 8
- On submit: call `authService.confirmPasswordReset(token, newPassword)`
  - On 400 error: show the server error message ("This link has expired or has already been used. Request a new password reset.") with a link to /forgot-password
  - On success: show "Password updated successfully! You can now log in." with link to /login
- Form: two password fields (new password + confirm password), submit button with loading state

```markdown
### İşlenen Kişisel Veriler

| Veri Alanı | Amaç | Saklama Süresi |
|------------|------|----------------|
| Ad Soyad (full_name) | Kullanıcı kimliği ve profil görüntüleme | Hesap silinene kadar |

| Şifre sıfırlama tokeni (SHA-256 hash) | Tek kullanımlık parola sıfırlama | 30 dakika geçerlilik, kullanım sonrası işaretlenir |
| Giriş başarısızlık sayacı | Güvenlik — hesap kilitleme (5 deneme) | Bellekte, sunucu yeniden başlayana veya başarılı girişe kadar |

### Veri Minimizasyonu

- Yalnızca sistem işlevleri için zorunlu veriler toplanmaktadır.
- Avatar yükleme isteğe bağlıdır; varsayılan avatar gerekmez.
- Şifre sıfırlama tokenleri ham değil, SHA-256 hash olarak saklanır.

### Kullanıcı Hakları

- **Erişim:** Kullanıcı kendi profil verilerine `GET /auth/me` ile erişebilir.
- **Düzeltme:** Kullanıcı profil bilgilerini `PUT /auth/me` ile güncelleyebilir.
- **Silme:** Kullanıcı hesabı silindiğinde ilgili veriler kaldırılır (soft delete; kalıcı silme admin yetkisi gerektirir).

- Şifreler bcrypt algoritması ile hashlenir; plaintext saklanmaz.
- JWT token'ları localStorage'da tutulmaktadır (v2'de HttpOnly cookie'ye geçiş planlanmaktadır).
```
