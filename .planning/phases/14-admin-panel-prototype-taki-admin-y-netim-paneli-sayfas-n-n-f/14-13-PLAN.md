Cluster A — fix the three "anchor-trigger downloads strip Authorization header → 401" gaps that block PDF (Test 5), CSV (Test 16), and JSON (Test 28) admin exports.

Closes 3 UAT gap-truths in one shot via a single shared authenticated-download helper. The existing `downloadCsv()` in `Frontend2/lib/admin/csv-export.ts` is a hidden-anchor `<a href>` click which never reaches the axios interceptor that injects the JWT bearer — so the browser fires GET /api/v1/admin/{users.csv,summary.pdf,audit.json} unauthenticated, backend `require_admin` rejects with 401, and the user sees a retry-icon download / 401 in network log.

Fix at the source: one shared helper that does `fetch(url, { headers: { Authorization: ... } })` → `.blob()` → `URL.createObjectURL` → programmatic anchor click → `URL.revokeObjectURL`.

Output: ONE new `download-authenticated.ts` helper + 3 consumer rewires + 1 unit test. Existing `downloadCsv` is either removed or restricted to public-only endpoints (no admin-only callers left).

    ```typescript
    // Phase 14 Plan 14-13 (Cluster A) — authenticated blob-download helper.
    //
    // Replaces the anchor-trigger downloadCsv() pattern for ANY admin-only
    // binary endpoint. The anchor-trigger pattern fires a plain GET that
    // bypasses the axios JWT interceptor, so backend require_admin returns
    // 401. This helper does the right thing: fetch with explicit
    // Authorization header → blob → object URL → programmatic click →
    // revoke.
    //
    // Single producer per Cluster A — consumed by:
    //   - app/(shell)/admin/layout.tsx (Rapor al → /admin/summary.pdf)
    //   - components/admin/users/users-toolbar.tsx (CSV → /admin/users.csv)
    //   - components/admin/audit/admin-audit-toolbar.tsx (JSON → /admin/audit.json)
    //
    // Token source: imports AUTH_TOKEN_KEY from @/lib/constants (mirrors the
    // axios interceptor at Frontend2/lib/api-client.ts:11-22). Applies the
    // SAME quoted-token guard from api-client.ts:18 — if localStorage holds a
    // JSON-stringified value, JSON.parse() unwraps it BEFORE the `Bearer `
    // prefix. Without this guard the backend receives `Bearer "<token>"`
    // (literal quotes) and returns 401.

    import { AUTH_TOKEN_KEY } from "@/lib/constants"

    export async function downloadAuthenticated(
      url: string,
      filename?: string,
    ): Promise<void> {
      if (typeof document === "undefined") return // SSR safety

      // Mirror Frontend2/lib/api-client.ts:14-19 exactly.
      let authHeader: Record<string, string> = {}
      if (typeof window !== "undefined") {
        const token = window.localStorage.getItem(AUTH_TOKEN_KEY)
        if (token) {
          // D-02 quoted-token guard (mirrors api-client.ts:18 verbatim).
          const cleanToken = token.startsWith('"') ? JSON.parse(token) : token
          authHeader = { Authorization: `Bearer ${cleanToken}` }
        }
      }

      const res = await fetch(url, {
        method: "GET",
        headers: authHeader,
        credentials: "include", // forward cookies if backend uses cookie auth in future (N-1: no-op without backend Access-Control-Allow-Credentials: true; verify before assuming this contributes anything)
      })

      if (!res.ok) {
        throw new Error(
          `downloadAuthenticated failed: ${res.status} ${res.statusText} for ${url}`,
        )
      }

      const blob = await res.blob()
      const objectUrl = URL.createObjectURL(blob)

      const a = document.createElement("a")
      a.href = objectUrl
      if (filename) a.download = filename
      a.style.display = "none"
      document.body.appendChild(a)
      a.click()
      document.body.removeChild(a)

      // Defer revoke by one tick so Firefox/Safari finish reading the URL.
      setTimeout(() => URL.revokeObjectURL(objectUrl), 0)
    }
    ```
