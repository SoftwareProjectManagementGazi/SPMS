<objective>
Cluster B — fix the "PM-scoped use cases reject admin destructive ops with 404" gap on project deletion (Test 23) AND audit the same risk family on archive / un-archive / UPDATE / any other admin-scoped destructive flow that delegates to PM-scoped use cases.

Closes UAT gap-truth: "Admin can delete any project from /admin/projects regardless of project ownership". Root cause: `Backend/app/application/use_cases/manage_projects.py:185-190` raises `ProjectNotFoundError` whenever `project.manager_id != manager_id`, even when the actor is a system admin. The endpoint catches this and returns 404 (info-disclosure-safe), so admins cannot delete projects they don't manage.

Per CLAUDE.md Clean Architecture (D-rule) and the GSD planner-authority constraint, the fix lives in the **use case** (where the business rule lives), NOT in the endpoint. The endpoint becomes a thin wrapper that passes `actor: User` instead of `manager_id: int`. The use case checks `actor.role` and bypasses the ownership rule when the actor is admin. Audit log records the admin-initiated delete with the original manager_id captured in metadata for compliance.

Purpose: Without this fix, /admin/projects "Sil" button is dead-functional — admins can't perform the destructive op the UI promises (D-B5 + D-B1: "All ~15 prototype actions = functional"). Same risk also affects UpdateProjectUseCase (UAT Test 22 / D-D2 imply admin should change ANY project's properties; if Update has the same guard, admin edits silently 404), archive, and un-archive. M-2 expansion: the audit list MUST include `UpdateProjectUseCase` and any other write-path use case in `manage_projects.py`.

Output: ONE use-case modification (admin-bypass on `DeleteProjectUseCase`) + endpoint signature update + new integration test set INCLUDING a mandatory sibling-flow test (M-1: skip-if-no-sibling, never silently omitted). If the audit pass discovers the same bug on archive/un-archive/update (Plan 14-05 reused these endpoints), this plan is split — see "Audit Findings" gate below.

    **Step 1 — Update `DeleteProjectUseCase.execute` signature to accept `actor: User`:**

    Following CLAUDE.md DIP discipline (use cases depend on Domain entities, NOT FastAPI primitives), the use case accepts the full User entity (not just an int). The use case then decides whether to bypass the ownership check:

    ```python
    # Backend/app/application/use_cases/manage_projects.py

    from app.domain.entities.user import User  # ABSTRACT — already domain-pure

    class DeleteProjectUseCase:
        def __init__(
            self,
            project_repo: IProjectRepository,
            audit_repo: IAuditRepository | None = None,  # optional for backward compat
        ):
            self.project_repo = project_repo
            self.audit_repo = audit_repo

        async def execute(self, project_id: int, actor: User) -> None:
            project = await self.project_repo.get_by_id(project_id)
            if project is None:
                raise ProjectNotFoundError(project_id)

            # Admin bypass — admins can delete any project regardless of ownership.
            # Per D-A6 + UAT Test 23 root cause: previously this raised
            # ProjectNotFoundError for any non-manager actor, blocking admin
            # ops on projects they don't personally manage.
            is_admin = (actor.role and actor.role.name.lower() == "admin")
            if not is_admin and project.manager_id != actor.id:
                raise ProjectNotFoundError(project_id)  # info-disclosure-safe 404

            original_manager_id = project.manager_id
            await self.project_repo.delete(project_id)

            # Compliance audit trail — ONLY when admin overrode the ownership
            # check (or always, depending on Phase 9 audit-log convention).
            # Plan 14-09 already wires create_with_metadata at all repo write
            # sites; this is an EXTRA admin-action entry on top of whatever
            # the repo already logs.
            if self.audit_repo is not None and is_admin and original_manager_id != actor.id:
                await self.audit_repo.create_with_metadata(
                    action="project.deleted_by_admin",
                    entity_type="project",
                    entity_id=project_id,
                    user_id=actor.id,
                    extra_metadata={
                        "target_manager_id": original_manager_id,
                        "project_key": project.key,
                        "project_name": project.name,
                    },
                )
    ```
