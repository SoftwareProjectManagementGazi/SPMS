<objective>
Cluster E — fix the Roles tab counts + cross-link gap (Test 19): role cards show literal "Kullanıcı: =" placeholder AND there is no working "Görüntüle" affordance to cross-link into the Users tab with the role filter pre-applied.

Two distinct sub-failures with shared roots:

1. **Counts render as "Kullanıcı: ="** — RoleCard's `userCount` binding receives a value that stringifies to `=` (likely `undefined` formatted by an i18n template that injects "{count}" but the {count} substitution is dropped). Inspecting `role-card.tsx` shows the count IS bound as `{userCount}` in JSX, which means `userCount` prop is being passed undefined / NaN. The most likely root: `roles/page.tsx` calls `useAdminUsers()` (Plan 14-03 hook) which returns the FIRST PAGE of users (limit 50 default) instead of ALL users — so `users.filter(u => u.role.name === 'admin').length` only counts admins on page 1. With 0 admins on page 1, the count is 0; with the expression somehow producing undefined (e.g., users array undefined during loading), the i18n format fallback shows `=`.

2. **Görüntüle is a no-op** — there is currently NO Görüntüle button in `role-card.tsx` at all (verified by file read; only "Kullanıcı: {n}" footer is present). The fix is to ADD the affordance + wire it to push `/admin/users?role=<id>` AND teach `/admin/users/page.tsx` to parse the `?role=` URL param on mount + seed the SegmentedControl.

Per CONTEXT D-A5, the prototype's "Düzenle" button was DELIBERATELY REMOVED from system-role cards (RBAC defer). The new "Görüntüle" affordance is a different, deliberate addition for cross-tab data consistency (D-W1) — it's a navigation aid, NOT a CRUD trigger, so it doesn't violate D-A5's removal rationale.

    **Step 2 — Add ?role= URL param parser to `app/(shell)/admin/users/page.tsx`:**

    ```typescript
    "use client"

    import { useSearchParams } from "next/navigation"

    // ... inside the page component, BEFORE the useLocalStoragePref:

    const searchParams = useSearchParams()
    const roleFromUrl = searchParams.get("role")  // null if not present

    // Compute the initial filter:
    //   - If ?role= is present, use that (and write through to localStorage in Step 3).
    //   - Else fall back to localStorage / DEFAULT_FILTER as today.
    const seededDefault: AdminUserListFilter = React.useMemo(() => {
      if (roleFromUrl) {
        return { ...DEFAULT_FILTER, role: roleFromUrl }
      }
      return DEFAULT_FILTER
    }, [roleFromUrl])

    const [filter, setFilter] = useLocalStoragePref<AdminUserListFilter>(
      FILTER_STORAGE_KEY,
      seededDefault,
    )

    // Effect: when ?role= appears AFTER mount (cross-tab navigation), force
    // the filter to update + write through to localStorage.
    React.useEffect(() => {
      if (roleFromUrl && roleFromUrl !== filter.role) {
        setFilter({ ...filter, role: roleFromUrl })
      }
      // Intentionally exclude `filter` from deps — only trigger on URL change.
    // eslint-disable-next-line react-hooks/exhaustive-deps
    }, [roleFromUrl])
    ```
