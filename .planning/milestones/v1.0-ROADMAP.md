# Milestone v1.0: MVP

**Status:** SHIPPED 2026-04-20
**Phases:** 1-7
**Total Plans:** 43

## Overview

SPMS v1.0 delivers a full-featured software project management platform. Starting from a brownfield Clean Architecture backend, the milestone fixed architecture violations, patched a critical RBAC bug, then built the complete feature set: authentication, team management, project/task CRUD with dependencies and recurrence, visual views (Kanban, Calendar, Gantt), notifications (in-app + email), reporting with export, and process model customization with external integrations.

## Milestone Summary

**Key Decisions:**
- Clean Architecture maintained throughout — use cases stay pure, infrastructure at edges
- Phase 1 bundled all ARCH + DATA + SEC fixes to eliminate blockers before feature work
- RBAC bug (ARCH-10) treated as Phase 1 blocker — no new endpoints shipped without authorization
- Soft-delete implemented at repository layer, not use case layer
- In-memory lockout store (intentional for dev — restarts clear state)
- frappe-gantt with dynamic import for SSR safety
- INotificationService abstraction enables WebSocket swap without touching callers
- Integration events use asyncio.create_task fire-and-forget pattern
- SystemConfigProvider wraps app inside AuthProvider for global config access

**Issues Resolved:**
- Critical RBAC authorization bug (any user could access any project's tasks)
- Dashboard/Settings/Activity showing mock data instead of live API
- JWT_SECRET and DB_PASSWORD hardcoded defaults
- WeasyPrint replaced with fpdf2 (pure Python, no system lib dependencies)
- Migration naming conflicts (004_phase5_schema.py starts with digit)

**Issues Deferred:**
- Phase 04/06 VERIFICATION.md human_needed items (UI overhaul planned for next milestone)

**Technical Debt Incurred:**
- In-memory lockout store (not persistent across restarts)
- Gantt dependency arrows deferred (ParentTask type lacks dependency IDs field)
- tests/integration/conftest.py uses main spms_db directly (not isolated test DB)
